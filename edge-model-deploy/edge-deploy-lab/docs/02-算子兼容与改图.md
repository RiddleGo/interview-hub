## 你要完成什么

确认 ONNX 在昇腾上 **能编过**；unsupported 算子做 graph surgery；建立 `operator_kb` 记录。

---

## 操作步骤

```bash
# 复制 FP32 为 adapt 版（无改图时直通）
bash scripts/06_atc_compile.sh adapt

# 算子预检（需 CANN）
bash scripts/06_atc_compile.sh check
```

查看 `deliverables/02_adapt/atc_mode1.log`，列出 unsupported / 受限算子。

用 Netron + `02_check_onnx.py` 的算子统计，对照芯片算子表。

---

## 若报 Unsupported（Q5 场景）

1. Netron 定位 **哪一个** 节点（Softmax 的 axis、前后接什么）。
2. PC 上换等价子图，`onnx.checker` + ORT 回归。
3. 替换 `deliverables/02_adapt/model_adapt.onnx`。
4. 写入 `operator_kb/ascend310_yolov5s.md`。

常见 YOLOv5 问题：`SiLU`、`Resize` 模式、大 kernel Conv——见知识库示例。
